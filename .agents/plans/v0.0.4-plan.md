# LazyDex v0.0.4-plan — Polish, Automation & Safety Nets

> **Tagline**: Turning a feature-rich app into a reliable daily driver.
> **Origin**: Items from the original `MVP_0.0.2-beta` plan that were deferred when the dex redesign (now `v0.0.2-plan`) took priority, plus new testing/infra needs discovered during v0.0.2 development.
> **Prerequisite**: Built on top of v0.0.2 codebase (dex redesign, genre/tag system, scraper sources, browser, DB migration v1→v2).

---

## 1. Automation & Background Tasks

### Auto-Backup (WorkManager)
- [ ] Scheduled backup — configurable: Never / Daily / Weekly
- [ ] SAF folder picker (`ACTION_OPEN_DOCUMENT_TREE`) — user selects destination directory
- [ ] Persisted URI via `takePersistableUriPermission`
- [ ] WorkManager `PeriodicWorkRequest` (min interval: Daily = 24h, Weekly = 168h)
- [ ] Timestamped filenames: `lazydex_backup_YYYY-MM-DD_HHmmss.lazydex`
- [ ] Safe copy flow: writes to temp file first, packages ZIP, copies to SAF only after validation
- [ ] Retention policy: keep last 30 backup files, auto-delete older
- [ ] UI feedback: last backup timestamp in Settings
- [ ] Error handling: failed backup → notification

**Repository:**
- [ ] `add()` — UUID generation, `normalize()` called, lastUpdated set
- [ ] `update()` — `normalize()` clamping, lastUpdated set
- [ ] `incrementProgress()` / `decrementProgress()` — auto-complete, auto-date, revert-on-decrement
- [ ] `replaceAll()` — atomic, normalize on each item, exception mapping
- [ ] `setStatus()` — auto-date logic for all paths (Plan To, Completed, On Hold, Dropped, Re-reading)

**Scraper:**
- [ ] Valid URL → parsed metadata
- [ ] Invalid URL → error
- [ ] Timeout → timeout Result via `withTimeout(30_000L)`
- [ ] Non-HTML response → Content-Type check error
- [ ] Large response → abort via `SizeLimitedSource`
- [ ] Auto-charset sniffing → correct parsing without mojibake
- [ ] SSRF SafeDns → local/loopback IPs blocked
- [ ] Source URL matching → correct source selected
- [ ] Source search → dispatches to correct source

**Backup:**
- [ ] Round-trip: serialize → deserialize → serialize → compare
- [ ] ZIP export packaging (backup.json + optional covers/)
- [ ] ZIP import extraction and parsing
- [ ] Schema version handling: missing→default 1, >2→rejected
- [ ] Edge cases: empty items, missing id, missing title, bad status, negative progress
- [ ] Merge: timestamps, local wins tie, pure-additions preserve timestamps
- [ ] Merge duplicate by URL: force local ID mapping
- [ ] v1→v2 merge field preservation (genres, tags, author, description, dates)
- [ ] All normalized after merge

**Domain:**
- [ ] Genre/tag normalization — dedup, underscore→space, casing
- [ ] Status adaptive labels per category
- [ ] Auto-date logic — all paths, manual override
- [ ] `UrlNormalizer` — lowercase, strip fragment, trailing slash, invalid URL

## 4. Safety & Error Handling Enhancements

### DB Corruption Recovery
- [ ] Detect `SQLiteDatabaseCorruptException` on DB access
- [ ] Error screen with message: "Database appears corrupted"
- [ ] [Reset Database] button — deletes DB, re-creates via `fallbackToDestructiveMigration()`
- [ ] Export current data before reset option

### Similar Title Detection
- [ ] On Save: check main title or alt titles fuzzy-match existing items
- [ ] Simple similarity: lowercase contains or Levenshtein distance < 3
- [ ] Warning: "Similar title already exists" — inline below Save button
- [ ] Non-blocking — user can still save

### Enhanced Validation
- [ ] URL format validation on sourceUrl (not just HTTPS prefix)
- [ ] Max title length: 200 chars
- [ ] Max notes length: 5000 chars
- [ ] Max alt titles count: 20

## Summary

**v0.0.4 is the layer of robustness, automation, and confidence** that turns the feature-rich v0.0.2 app into a reliable daily driver. Most of these items are non-visual — tests, safety nets, build tooling, and background automation. The visible UX polish (animations, rating colors, i18n) is secondary.

**Existing v0.0.2 features this depends on**: Source interface, AniListSource, BackupProcessor v2, DB migration v1→v2, DexViewModel, genre/tag system.
